# 仕様変更時の影響範囲特定課題
## **【対象システム】**
社内図書管理システム（Webアプリ）

## *【現状の仕様】**
* ユーザーは一度に最大 **5冊** まで本を借りることができる。
* 返却期限を過ぎた本が1冊でもあるユーザーは、新規の貸出ができない。

## **【仕様変更要求】**
「福利厚生の充実のため、貸出上限冊数を **10冊** に引き上げたい。ただし、システム負荷軽減のため、貸出履歴の保存期間を無制限から **過去3年間** に変更する。」

| 分析レイヤー | 影響が予想される箇所（画面・機能・データ） | 懸念されるリスク / テストすべき内容 |
| --- | --- | --- |
| **① 表面的な影響** | (例) 貸出ボタンのバリデーション | 10冊までボタンが活性化するか、11冊目でエラーが出るか |
|  |  |  |
| **② 構造的な影響** |  |  |
|  |  |  |
| **③ 非機能的な影響** |  |  |

## **回答例**

- **① 表面的な影響**
  - 貸し出し画面
    - 最大5冊の表記
      - 6冊以上のレイアウト
        - ページネーション
        - デザイン崩れ等
    - エラーメッセージ(１０冊以上への変更)

- **② 構造的な影響**
  - DBで返却、貸出処理
    - パフォーマンスが劣化
    - 返却期限を過ぎた本が1冊でもあるユーザーは、新規の貸出ができない。　
      - (再帰テスト)→3年以上の場合？を追加
  - 貸し出し履歴で3年以上
    - 現在貸出中のデータは削除されないか？
    - 3年以上削除処理の追加(データ適応バッチ)
- **③ 非機能的な影響**
  - 一人当たりの貸し出しによるデータパフォーマンスが低下しないか
  - 容量不足にならないか？
  - バックアップ
    - 3年以上のデータのバックアップは？
